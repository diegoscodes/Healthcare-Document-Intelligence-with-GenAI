Project Name: MediRAG

Healthcare Document Intelligence with GenAI

From complex healthcare documents to structured, actionable, auditable data.

---

## Project Overview

### What is MediRAG?
MediRAG is a GenAI-based Document Intelligence system designed to process, understand, and structure complex healthcare and pharmaceutical documents such as:
- Pharmacy Agreements
- Prior Authorizations
- Clinical and Administrative Healthcare Documents

It combines Large Language Models (LLMs) with Retrieval-Augmented Generation (RAG) to transform unstructured documents (PDFs, long-form text) into structured, reliable, and auditable data.

---

## Current Status (Backend MVP)
Implemented and working today:
- Professional backend API (FastAPI) with documented endpoints
- PostgreSQL persistence (SQLAlchemy + Alembic migrations)
- PDF upload + file persistence
- Per-page text extraction and storage (evidence layer)
- Document serving (inline preview + download)
- Semantic chunk indexing to Weaviate (DocumentChunk collection)
- Agentic QA endpoint with an explicit workflow:
  - Planner (multi-query strategy)
  - Retriever (Weaviate vector search + metadata filtering by document_id)
  - Generator (LLM returning structured JSON: answer + citations)
  - Verifier (groundedness checks + citation validation)
  - Trace steps returned for audit/debug
  - Evaluation mode: allow_insufficient=false to disallow refusal responses
- Automated tests for verifier behavior (groundedness + eval mode)
